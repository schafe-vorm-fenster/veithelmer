# Project Guidelines

## Content Architecture

### Localization and File Naming
All content bundles must support bilingual localization (German/English).
To ensure consistency across the static site generator, use the following file naming convention for all localized Markdown files:

- **German Content:** `index_de.md`
- **English Content:** `index_en.md`

These files must coexist within the same content directory (e.g., `/content/films/[film-slug]/`).

**Example Structure:**
```text
/content/films/absurdistan/
├── index_de.md  (German metadata and content)
├── index_en.md  (English metadata and content)
└── ...
```
